### 目录结构

- `assets`：存放资源的目录，与B站上的教程一致
- `src`：根据章节和小节写下的代码，问题可能是前半程未确定好自己的目录结构，会有很多有问题，如果遇到了麻烦，可以及时反馈
- `third_party`：第三方库的头文件、静态库、动态库、源文件
  - `include`：头文件
  - `lib`：Windows环境下编译好的动态库（需设置环境变量），静态库
  - `src`：一些库的源文件，由CMake组织并编译成静态库

### 注意事项

1. assimp要自己设置环境变量，Windows我确实没找到方法如何让CMake编译的程序能找到对应的动态库文件
   - 除了设置环境变量，还有种方式就是使用CMake，每次配置目标时，把dll文件复制到对应exe目录下，我嫌占用资源，就没这么干了
   - 该方式可参考我`1_05_Shader_Exercise`复制Shader源文件的代码
2. 这个项目我是在两台电脑上进行学习的，如果把assimp环境变量设置好，应该在Windows之间使用VS+CMake没什么问题，GCC+CMake还没有使用过
3. 资源路径和着色器路径是让CMake传递对应宏，来进行字符串拼接的。但前面一开始没有这么做，一开始是直接将资源复制到exe所在目录来实现的
4. 有些代码可能会有问题，因为文件结构修改过很多次，有些代码后面修正了，但是前面没有
   - 例如：按`L`解锁鼠标，后面是使用回调函数，而一开始则直接是写入`processInput()`函数中，导致按`L`并不能保证只执行一次
5. 有些文件进行了修改，比如摄像机类，`WSAD`只能在XOZ平面上移动，按`Space`和`LCtrl`才能上升或下降
6. `ReplaceText.py`是因为摄像机类把函数进行了修改，因此直接使用Python脚本进行了文本替换
